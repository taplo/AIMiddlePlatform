# AI算法调度中台 — 设计文档

日期: 2026-07-08
状态: 草稿

## 1. 概述

### 1.1 项目目标

构建一个大小模型协同的计算机视觉 AI 算法调度中台，集成 14+ 种 CV 算法能力，以 LLM Agent 为决策大脑、专用小模型集群为高频推理引擎，对外提供统一的算法服务 API，对内实现模型注册→编排调度→结果输出的全链路管理。

### 1.2 核心模式

大小模型协同采用 A+B+C 三阶段演进：

| 阶段 | 模式 | 说明 |
|----|------|------|
| A | 大模型决策+小模型执行 | LLM Agent 理解场景，动态选择/编排小模型 |
| B | 小模型高频+大模型兜底 | 小模型处理常规情况，低置信度转大模型 |
| C | 知识蒸馏 | 用大模型高质量输出蒸馏提升小模型精度（未来） |

### 1.3 部署场景

- **公有云/私有云中心**：集中部署，支持大规模扩展
- **私有化部署**：全内网，支持国产 GPU（昇腾/寒武纪）
- 两种部署共享同一套架构和代码

### 1.4 目标用户

- **算法工程师**：模型注册、版本管理、效果评估、监控
- **应用开发者**：通过 REST API/WebSocket/SDK 调用算法能力

## 2. 需求与约束

### 2.1 业务场景

首期以安防监控为主，涵盖但不限于：
- 目标检测与识别（人、车、物）
- 人脸识别/行人重识别 (ReID)
- 车牌识别/车辆检测/车型识别
- 文字识别 (OCR)
- 横幅/垃圾箱等特定目标识别
- 人群聚集/异常事件检测
- 行为识别/轨迹分析（需视频时序上下文）

### 2.2 规模约束

| 维度 | 当前 | 远期 |
|----|------|------|
| 摄像头路数 | 1,000+ | 10,000+ |
| 延迟要求 | 准实时 (<1s 快速路径) | 同上 |
| 推理类型 | 单帧 + 离线批量 | 增加视频流算法 |
| 大模型 | Qwen-VL / DeepSeek-VL | 按需替换 |

## 3. 系统架构

### 3.1 整体架构

采用 LLM Agent + 工具调用架构，融合固定 DAG 快速路径和场景预处理器，共 7 个核心模块：

```
┌─────────────────────────────────────────────────────────────┐
│                    管理控制台 (Web UI)                        │
│  模型管理 / 监控大屏 / 场景配置 / 告警管理                    │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                      API 网关                                │
│      REST API / WebSocket / SDK / 认证限流                   │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                      LLM Agent 决策大脑                       │
│       场景理解 → 工具编排 → 兜底分析 → 结果合成              │
└──────────┬─────────────────────────────────────┬────────────┘
           │                                      │
┌──────────▼────────────┐          ┌──────────────▼──────────┐
│   工具注册中心          │          │    推理编排器             │
│   (Tool Registry)      │          │  并行/串行/条件调度       │
└──────────┬────────────┘          └──────────────┬──────────┘
           │                                      │
┌──────────▼──────────────────────────────────────▼──────────┐
│                   小模型推理集群                             │
│  目标检测 | 人脸识别 | 车牌识别 | 车辆检测 | ReID | OCR...  │
│  (每个模型独立部署，独立扩缩容)                             │
└──────────┬───────────────────────────────────────────────┬─┘
           │                                                │
┌──────────▼──────────────┐      ┌─────────────────────────▼─┐
│   视频流接入层            │      │   ★ 视频缓存服务             │
│   RTSP/GB28181 → 抽帧    │      │   环形缓冲(5~60s)           │
│   → ROI → 帧队列         │      │   支持视频段按需提取          │
└──────────────────────────┘      └───────────────────────────┘
```

### 3.2 三路径设计

系统设计了三条处理路径，按场景类型自动路由：

#### 快速路径 (Fast Path) — DAG 固定管线

- **流量占比**: 80-90%
- **延迟**: 100-300ms
- **适用场景**: 车牌识别、人脸通行、结构化占道等规则固定的成熟业务
- **工作原理**: 场景路由器匹配已知场景哈希 → 加载预定义 DAG → 小模型推理 → 直接输出

#### Agent 路径 (Agent Path) — LLM 灵活决策

- **流量占比**: 5-15%
- **延迟**: 1-5s
- **适用场景**: 异常事件研判、跨场景关联分析、新场景适配
- **工作原理**: 场景预处理器(万物识别) → LLM Agent 理解/推理/编排 → 推理调度 → 大模型兜底 → 结果合成

#### 视频算法路径 — 时序上下文感知

- **流量占比**: &lt;5%
- **适用场景**: 行为识别、轨迹分析、动作检测等需多帧上下文的算法
- **工作原理**: 从视频缓存服务提取片段 → 视频级算法处理 → 输出

### 3.3 场景路由器

```
请求入站 → 提取场景特征 (场景ID/算法类型/视频源)
  → 查场景路由表 (内存哈希表)
    ├─ 命中 → 直接执行对应 DAG 管线
    └─ 未命中 → 转发到 Agent 路径
```

- 路由表支持热加载
- 命中率监控，自动识别可固化的场景模式
- 降级策略：Agent 路径故障时全量切换 DAG 路径

### 3.4 场景预处理器

在 LLM Agent 之前运行轻量开放词汇检测模型（YOLO-World / Grounding DINO）：

```
输入帧 → 预处理器
  → 结构化信息 (场景标签 + 目标列表 + 位置 + 统计)
  → 注入 LLM Agent 的 System Prompt
```

减少 LLM 的 Token 消耗和视觉幻觉。

### 3.5 工具注册中心 (Tool Registry)

每个小模型注册为一个"工具"：

| 字段 | 说明 |
|----|------|
| tool_id | 唯一标识 |
| name | 模型名称 |
| description | 自然语言描述（LLM 理解用途） |
| input_schema | 输入参数 Schema |
| output_schema | 输出格式 Schema |
| cost_estimate | 预估计算成本/延迟 |
| status | online/offline/deprecated |
| version | 版本号 |

### 3.6 推理编排器

- 快速路径：按 DAG 定义直接编排
- Agent 路径：按 LLM 决定的工具列表调度
- 支持并行调用、超时管理、指数退避重试
- 结果缓存（相同场景/帧去重）

## 4. API 设计

### 4.1 对外 API（面向应用开发者）

```
POST /v1/analyze/frame          — 单帧/图片分析
POST /v1/analyze/stream         — 注册视频流分析任务
GET  /v1/tasks/{task_id}/results — 拉取结果
WS   /v1/ws/stream/{task_id}    — WebSocket 实时结果
```

### 4.2 管理 API（面向算法工程师）

```
POST /v1/models                 — 注册新模型
POST /v1/models/{id}/deploy     — 部署/更新模型
POST /v1/pipelines              — 注册快速路径 DAG
GET  /v1/monitor/overview       — 全局大盘
GET  /v1/monitor/models         — 各模型指标
GET  /v1/monitor/streams        — 各视频流状态
```

## 5. 部署架构

### 5.1 云中心部署

- 小模型推理集群：每个模型独立 K8s Deployment，HPA 自动扩缩容
- LLM 推理集群：vLLM 多节点部署，GPU 节点池
- 核心控制面：场景路由器 / Agent 服务 / 推理编排器，2-3 副本
- 视频接入服务：弹性扩容

### 5.2 1000→10000 路伸缩路径

| 阶段 | 摄像头数 | 规模 | 关键配置 |
|----|---------|-----|---------|
| Phase 1 | 1,000 路 | 8-16 GPU | 小模型 30ms/帧，大模型 5% 流量 |
| Phase 2 | 5,000 路 | 32-64 GPU | 小模型按算法分片，Redis 缓存 |
| Phase 3 | 10,000+ 路 | 多集群/多机房 | 区域化部署，消息队列解耦 |

### 5.3 私有化部署差异

- 完全内网，无需互联网访问
- 私有镜像仓库 + 离线模型包
- 支持国产 GPU（昇腾/寒武纪）
- Helm Chart 一键部署

## 6. 错误处理与可靠性

| 故障场景 | 处理策略 |
|---------|---------|
| 小模型超时/异常 | 熔断 + 自动转 Agent 路径 |
| LLM 服务不可用 | 全量降级到 DAG 路径 |
| 结果断言失败 | 重试 2 次（指数退避）→ 标记异常 |
| 视频流断开 | 自动重连 + 告警 |
| 帧积压 | 动态降帧率 + 告警 |

## 7. 管理控制台

### 7.1 模型管理

- 注册/版本管理/蓝绿部署/回滚
- 在线/离线状态切换
- 性能指标：延迟 p50/p95/p99、吞吐量、错误率
- 成本追踪

### 7.2 场景配置

- DAG 快速路径可视化编排
- 场景路由规则管理
- 置信度阈值调优

### 7.3 监控告警

- 全局实时大盘：帧率、活跃流、路径占比
- 模型延迟/错误率/GPU 告警
- 视频流中断告警
- 通知渠道：钉钉/企微/飞书
